Passmark TestOps Guide

Tài liệu tổng quan để người mới hiểu mục đích hệ thống, cách vận hành, flow chạy test, API và các nguyên tắc phát triển tiếp.

Runtime localhost:5000 Database PostgreSQL + Prisma Automation Playwright Chromium
Mục đích Kiến trúc Flow sử dụng Run queue API Cài đặt Nguyên tắc Xử lý lỗi

1. Mục đích hệ thống

Passmark TestOps là AI Automation Studio chạy local, giúp QC/Tester tạo project, quản lý target, quản lý suite/case và chạy automation test qua UI thay vì phải tự viết Playwright thủ công.

Nguyên tắc lõi

Với SEO Basic, AI chỉ tạo test plan JSON. Backend validate plan và render Playwright code từ template ổn định, không để AI sinh raw TypeScript cho flow chính.

2. Kiến trúc tổng quan

Backend

  • Node.js + TypeScript, native HTTP server trong src/server.ts.
  • Prisma làm ORM, PostgreSQL là database chính.
  • Local AI client nằm tại src/local-ai-client.ts và chỉ đọc cấu hình từ .env.
  • Run worker dùng in-memory queue, concurrency mặc định bằng 1.

Frontend

  • Static HTML/CSS/JS trong public/.
  • UI dashboard tối màu, hỗ trợ VI/EN qua public/i18n/.
  • Frontend gọi backend API, không gọi local AI trực tiếp.
  • Run Center poll kết quả qua GET /api/runs/:id.
UI Backend API Queue Worker Playwright PostgreSQL

3. Flow sử dụng cho QC/Tester

  1. Tạo Project: nhập tên, môi trường và Base URL mặc định.
  2. Tạo Test Target: khai báo nơi cần test như web-url, local-web, source-code hoặc api.
  3. Tạo Test Suite: chọn loại suite, hiện tại flow chính là SEO Basic.
  4. Quản lý Test Case Library: bật/tắt case, sửa mô tả, priority và expected result.
  5. Chạy test: chọn project, suite, target, nhập AI request nếu cần rồi bấm Run test.
  6. Xem kết quả: theo dõi queued/running, xem Latest Result, AI Rendered Flow, code Playwright và Run History.

4. Test Target Management

Type Mục đích Dữ liệu chính
web-url Test website public hoặc staging. url
local-web Test app local đang chạy trên máy. url, ví dụ http://localhost:3000
api Lưu endpoint/base URL cho API testing phase sau. url, config
source-code Lưu đường dẫn source để source scan phase sau. localPath, config

Nếu không chọn target khi chạy test, hệ thống fallback về project.baseUrl để không phá flow cũ.

5. Run Test Flow và Queue

  1. User bấm Run test.
  2. Backend tạo TestRun với status queued và trả runId ngay.
  3. Frontend hiển thị trạng thái queued/running và poll /api/runs/:id.
  4. Worker chuyển run sang running, gọi AI nếu cần, render spec và chạy Playwright Chromium.
  5. Backend lưu TestResult, Artifact, generated code, stdout/stderr.
  6. Run kết thúc với passed, failed hoặc cancelled.
Server restart

Khi backend start lại, các run cũ còn queued/running được chuyển sang cancelled để trạng thái không bị treo.

6. SEO Basic Cases mặc định

SEO-001 Page loads successfully SEO-002 Title exists SEO-003 Meta description exists SEO-004 Canonical URL exists SEO-005 Exactly one H1 exists SEO-006 HTML lang exists SEO-007 Viewport meta exists

Run flow chỉ chạy các test case đang enabled. Riêng SEO Basic vẫn giữ đủ enabled cases từ thư viện case, không để AI cắt bớt.

8. Hướng dẫn chạy local

docker compose up -d postgres
npm install
npm run db:generate
npm run db:migrate:dev
npm run db:seed
npm run web

Mở hệ thống tại http://localhost:5000.

Biến môi trường cần có

PORT=5000
DATABASE_URL=postgresql://passmark:passmark@localhost:5432/passmark
LOCAL_AI_BASE_URL=https://your-local-ai-host/v1
LOCAL_AI_API_KEY=your-local-ai-api-key
LOCAL_AI_MODEL=your-local-ai-model

LOCAL_AI_BASE_URL chỉ để base /v1, không thêm /chat/completions.

9. Nguyên tắc phát triển tiếp

  • Không đổi format API response nếu không thật sự cần.
  • Không để frontend gọi local AI trực tiếp.
  • Không hardcode URL/key/model AI trong source code; luôn lấy từ .env.
  • Không quay lại SQLite cho flow chính; SQLite chỉ là legacy migration.
  • Không để AI generate raw TypeScript cho SEO Basic.
  • Giữ template renderer là nguồn tạo Playwright code ổn định.
  • Queue hiện là in-memory, thiết kế để sau này nâng cấp Redis/BullMQ.

10. Xử lý lỗi thường gặp

Docker daemon chưa chạy

Nếu gặp lỗi open //./pipe/docker_engine, mở Docker Desktop trước rồi chạy lại docker compose up -d postgres.

Prisma EPERM trên Windows

Tắt server/node đang giữ Prisma engine rồi chạy lại npm run db:generate.

Local AI lỗi cấu hình

Kiểm tra LOCAL_AI_BASE_URL, LOCAL_AI_API_KEY, LOCAL_AI_MODEL trong .env.

Playwright thiếu browser

Chạy npx playwright install chromium nếu máy chưa có Chromium runtime.

11. Roadmap gợi ý

  • API testing runner cho target type api.
  • Source scan cho target type source-code.
  • Redis/BullMQ cho queue production.
  • Multi-user, phân quyền, audit log.
  • Export report PDF/DOCX và dashboard xu hướng lỗi.
